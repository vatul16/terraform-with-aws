🚀 **Progress Update: Mastering Infrastructure as Code with Terraform!**

I’m excited to share that I’ve successfully completed **Day 3** of my *30 Days of AWS Terraform Journey*!

Today marked a shift from theory to hands-on practice as I provisioned my **first cloud resource using Infrastructure as Code (IaC)**.

Here’s what I accomplished:

🔹 **Resource Provisioning**
Defined and deployed an AWS S3 bucket using a Terraform configuration file (`main.tf`), turning code into real infrastructure.

🔹 **The Terraform Lifecycle**
Gained practical experience with the core workflow:

* `terraform init` — Initialize providers
* `terraform plan` — Preview changes safely
* `terraform apply` — Provision infrastructure in AWS

🔹 **State Management**
Explored how Terraform’s state file tracks resources, enabling efficient updates (like modifying bucket tags) without recreating the entire resource.

🔹 **Clean-Up & Cost Awareness**
Learned to use `terraform destroy` to remove resources when no longer needed—an essential practice for maintaining cost efficiency.

This hands-on experience reinforced how powerful IaC is for building scalable, consistent, and repeatable cloud environments.

Grateful for the guidance and mentorship throughout this journey—looking forward to tackling more advanced concepts in the coming days!
