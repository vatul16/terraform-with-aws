🚀 **Day 2 of My 30-Day Terraform Learning Journey**

Excited to share that I’ve successfully completed Day 2, diving into one of the most essential pillars of Infrastructure as Code: **Terraform Providers**.

Here’s a quick snapshot of what I explored today:

🔹 **What are Providers?**
Providers act as the bridge between Terraform’s configuration (HCL) and cloud platforms like AWS, translating code into real infrastructure actions.

🔹 **Why Versioning Matters**
I learned the importance of locking both Terraform and provider versions. Version constraints ensure stability, consistency, and help avoid unexpected issues in production environments.

🔹 **Hands-on with Documentation & Implementation**
Navigating the Terraform Registry is a key skill. Today, I practiced identifying the right provider configurations and even wrote my first resource block to create an AWS VPC.

🔹 **Authentication & Core Workflow**
Set up secure AWS authentication using IAM credentials and worked through Terraform’s core workflow:

* `terraform init` — Initialize and download providers
* `terraform plan` — Validate changes before applying

These fundamentals are crucial for building scalable and reliable cloud infrastructure.
