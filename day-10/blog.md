## 🧠 Why Terraform Expressions Matter

Writing Terraform code is one thing…
Writing **smart and scalable code** is another 🔥

Expressions allow us to:
✔️ Reduce code duplication
✔️ Add logic to configurations
✔️ Build dynamic infrastructure

---

## 🔑 Key Concepts I Learned

### 🔹 Conditional Expressions → Smart Decisions ⚖️

Using the ternary operator (`? :`), I can now dynamically configure infrastructure based on environments.

💡 Example:

* `t2.micro` for dev
* `t3.micro` for prod

➡️ All handled in a single line of code!

---

### 🔹 Dynamic Blocks → Clean & Scalable Code 🧱

This was my highlight of the day 🚀

Instead of writing repetitive nested blocks (like security group rules), I can now:
✔️ Use `dynamic` + `for_each`
✔️ Generate blocks automatically
✔️ Keep configurations clean and maintainable

---

### 🔹 Splat Expressions → Simplified Data Access ✨

Using the `*` operator, I can extract attributes from multiple resources easily.

💡 Example:

* Get all instance IDs
* Collect IP addresses into a list

➡️ Perfect for outputs and chaining resources

---

Today’s biggest realization:
